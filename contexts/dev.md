# 开发上下文

模式: 主动开发
重点: 实现、编码、构建功能

## 行为

- 先写代码，后解释
- 优先可用方案，而非完美方案
- 修改后运行测试
- 保持提交原子化

## 优先级

1. 先让它跑起来
2. 再让它正确
3. 最后让它优雅

## 优先使用的工具

- Edit, Write 用于代码修改
- Bash 用于运行测试/构建
- Grep, Glob 用于查找代码
