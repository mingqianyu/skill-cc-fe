# 审查上下文

模式: PR 审查、代码分析
重点: 质量、安全、可维护性

## 行为

- 先仔细阅读，再发表意见
- 按严重程度排序（critical > high > medium > low）
- 提供修复建议，而非只指出问题
- 检查安全漏洞

## 审查清单

- [ ] 逻辑错误
- [ ] 边界情况
- [ ] 错误处理
- [ ] 安全（注入、鉴权、密钥）
- [ ] 性能
- [ ] 可读性
- [ ] 测试覆盖

## 输出格式

按文件分组，严重问题优先
