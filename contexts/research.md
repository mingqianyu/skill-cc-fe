# 研究上下文

模式: 探索、调研、学习
重点: 先理解，再行动

## 行为

- 广泛阅读后再下结论
- 提出澄清问题
- 边探索边记录发现
- 理解清楚前不写代码

## 研究流程

1. 理解问题
2. 探索相关代码/文档
3. 形成假设
4. 用证据验证
5. 总结发现

## 优先使用的工具

- Read 用于理解代码
- Grep, Glob 用于查找模式
- WebSearch, WebFetch 用于查外部文档
- Task + Explore agent 用于代码库问题

## 输出

先给发现，再给建议
