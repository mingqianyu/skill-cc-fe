# /learn 命令

分析当前会话，提取可复用的模式并保存为技能文件。

## 使用时机

在会话中解决了非平凡问题后运行 `/learn`。

## 提取内容

1. **错误解决模式** - 什么错误、根因、修复方法
2. **调试技巧** - 非显而易见的调试步骤
3. **变通方案** - 库的怪癖、API 限制
4. **项目特定模式** - 代码库约定、架构决策

## 输出格式

保存到 `~/.claude/skills/learned/[模式名].md`：

```markdown
# [描述性模式名称]

**提取时间:** [日期]
**适用场景:** [简述]

## 问题
[解决什么问题]

## 解决方案
[模式/技巧/变通方案]

## 示例
[代码示例]

## 使用时机
[触发条件]
```

## 注意事项

- 不提取简单修复（拼写错误、语法错误）
- 不提取一次性问题
- 聚焦可复用的模式
- 一个 skill 只包含一个模式
